Chapter 2

Ruby Data Types

Data is the key to all applications, and there's not a lot you can do on a Web site without it. Before you can start creating a Web application, you need to know what data it will require and how you're going to work with it. In this chapter, we'll explore how Ruby handles data.

Working With Numbers

Just about every application uses numbers. Whether you want to calculate complex mathematical equations or keep track of your bowling scores, numbers will often play a crucial part in your Ruby code. We'll be using numbers to identify the postings in our restaurant blog. It helps to use numbers for this, because two records could have the same title. Assigning a unique number will allow us to easily retrieve the correct data record.

Ruby divides numbers into two categories:

  • Integer values

  • Floating-point values

If you've done programming before, this probably doesn't surprise you. If this is your first taste of programming, you might not have realized that there are two types of numbers. Let's take a closer look at these and see what's up.

Integer Values

The most basic way of handling numbers is with integer values. The integer value data type consists of only whole numbers, both positive and negative. These values represent numbers that don't have fractions, such as 1 dog, 1,000 people, or -20 degrees.

We use integer values in all sorts of applications. This data type is commonly used for counting (such as determining how many items are in a database table) and providing an identification number to records (such as an employee ID).

The Ruby integer data type does have one oddity that you should be aware of. In the written English language, large numbers are sometimes separated using commas, such as 10,236,431. This helps you to process the number in your mind easily.

In Ruby, you can't use commas in an integer value. In most mathematical operations, you just use the number without commas: 10236431. However, Ruby does allow you to use underscores instead of commas: 10_236_431. This format may look odd, but it can come in handy when dealing with really big numbers.

Unlike some other programming languages, Ruby puts no practical limit on number sizes. Ruby supports integer values using two methods. The fixnum method directly assigns the integer value to memory in binary form. These values can be as large as your operating system allows (2^30 for 32-bit operating systems, and 2^62 for 64-bit operating systems).

However, Ruby doesn't stop there. It also provides the bignum method, which can use multiple fixnums to store a larger integer value automatically, with no action required on your part! The bignum method can operate with extremely large numbers with ease!

Floating-Point Values

In contrast to integer values, floating-point values can represent any type of number, positive or negative, whole or fraction. You can't easily work with fractions in programming code, so Ruby uses decimal notation to represent fractions.

Decimal notation uses a decimal point to separate the whole-number portion of a value from the fraction portion. Thus, to represent 4 1/2, you would use 4.5, with the 5 representing 5 tenths, which is one half.

The floating-point data type provides for a wide assortment of numerical values, such as:

  • 1.25

  • 0.12345

  • -48.344

  • 3.245e2

You may notice something odd about the last example. This format is called scientific notation. If you're a math geek, you probably know just what this does. If not, the e represents the exponent, or multiplier. Don't confuse this with the mathematical constant e (in fact, you can also use E for the exponent symbol to avoid confusion).

The scientific notation multiplies the coefficient (the first number) by 10 raised to the exponent value. If you remember back to your early math classes (which were quite a while ago for some of us), multiplying by 10 just means moving the decimal point to the right.

So after all of that, the floating-point value 3.24e2 is just a fancy way of writing 3.24 times 100, which is 324.5!

Using Math

Ruby provides special symbols to allow you to perform arithmetic. It supports all of the basic mathematical operations, although you may not recognize some of the symbols it uses.

Ruby Mathematical Symbols
Symbol Operation
+ Addition
- Subtraction
* Multiplication
/ Division
% Modulo (remainder)
** Exponent

When completing mathematical operations with integer values, it's important to remember that the result will always be another integer value. Thus, if you try the operation 5 / 3, you'll get an answer of 1. You must use the modulo symbol to determine if there is a remainder. For example, the answer to 5 % 3 is 2 (2 is the remainder when you divide 5 by 3).

With floating-point arithmetic, the answer will always be a floating-point value. For example, performing the operation 1.0 + 2.0 will result in the answer 3.0.

Strings

So far, you've seen how to deal with numbers, but numbers aren't the only data element used in programs. Somehow you need to be able to store and manipulate words. That's where strings come in.

Strings in Ruby, unlike the ones you see in an orchestra, are values that contain text. A string usually contains printable characters, but it doesn't have to. It can also contain nonprintable control characters, such as tabs and return characters.

You create strings using literals. A literal is a sequence of characters. However, you need to let Ruby know where the sequence of characters begins and ends. You do this using string delimiters.

There are two basic types of delimiters: the single quote and the double quote. You can use these basic characters to indicate the beginning and end of the string:

"This is a sample string"
'This is another sample string'

But what do you do when you have to use a single or double quote inside a string value? The simple solution is to use the opposite quote to delineate the string:

"This isn't a problem in Ruby"
'Rich said, "This will work."'

However, there are times when you can't avoid using the same quote within the string. In these situations, you can use the backslash character to indicate that the quote is part of the string, not the delimiter:

'This isn\'t a problem'
"I replied, \"Rich said, ‘This'll work, too.' And he was right.\" "

One confusing feature of strings is that they can contain numbers. For example, you can have the string:

"12345"

While this string contains only numbers, you can't use it in a mathematical operation. Ruby handles strings differently from integers and floating-point numbers. There is no numerical value to the string; it's just a group of characters.

That covers the basic data types you'll run across in Ruby. In the next chapter, we'll look at how to store them in your programs.



Chapter 3

Using Variables

Trying to write a program using just the Ruby data types can be tricky, if not impossible. Once you calculate an answer from one equation, you often need to use it later in another equation. It's helpful to have the ability to store this information in your program. Ruby provides this feature by using variables.

Variables are words that represent a location in memory where Ruby stores a value. You can then reference the word anywhere else in your Ruby program to recall the value you originally placed there. You can also replace the value in the variable. Ruby just places the new value in the memory location and discards the original one.

In our Restaurant Review project, we'll use variables to pass data between our Ruby programs and our Web pages and to retrieve information from the database. Variables are a necessity in any program, including Ruby programs.

A variable in Ruby can be any combination of letters and numbers, but it must start with a lowercase letter. To place a value in a variable, you use an expression. An expression assigns a result or a value to a variable. Here are some examples of simple Ruby expressions:

test1 = 3
test2 = 45.36
testString = "This is a test string"
result = test1 + test2

The first expression assigns the integer value 3 to the variable named test1. The second expression does the same thing, but with a floating-point value. The third expression assigns a string value to the variable. The last expression performs a mathematical operation and assigns the result to the variable result.

Variables in Action

It's hard to demonstrate variables in Ruby without seeing them in action. Let's fire up a Ruby interpreter and start playing with variables.

We can do that using the Interactive Ruby program (called IRB). First, we need to get to a Command Prompt window (or Terminal window if you're using Mac or Linux) that supports our Ruby environment. For the InstantRails package, here are the steps for doing that:

  1. Start the Windows Explorer program on your PC.

  2. Double-click the folders until you reach the folder where you installed the InstantRails program (in the Lesson 1 example, I recommended c:\rails).

  3. Double-click the InstantRails.exe file.

You should now see the InstantRails control window.



The InstantRails control window

The InstantRails control window allows you to start a Command Prompt session configured to support the Ruby programming environment. To start that, just follow these steps:

  1. Click the I icon at the top-left corner of the InstantRails control window.

  2. Move your mouse pointer down to Rails Applications. A new submenu should appear.

  3. On the submenu, select Open Ruby Console Window.

You should get a command prompt that looks like this:

c:\rails\rails_apps>

You're now ready to use the IRB program. Type the command irb at the command prompt, and hit the ENTER key. You should see:

C:\rails\rails_apps>irb
irb(main):001:0>

The IRB program prompt appears, waiting for you to enter some Ruby statements. It shows the line number in the IRB session (001), along with the code program being executed (main). Let's begin by assigning some values to variables. Follow along with this example in your IRB session:

irb(main):001:0> test1 = 3
=> 3
irb(main):002:0> test2 = 45.36
=> 45.36
irb(main):003:0> test3 = test1 + test2
=> 48.36
irb(main):004:0>

The IRB program displays the result of each statement as you type it on the line. It places the => symbol on the line to indicate that the result is an internal value that Ruby processes from the statement. It's not what you would see in the program output.

To display a value from Ruby, use the puts command. The puts command displays the result of an assignment or the value of a variable. Type this command in your IRB session:

irb(main):004:0> puts test3
48.36
=> nil
irb(main):005:0>

IRB first shows the output of the puts command—the current value stored in the test3 variable. It then displays the result of the puts command, using the => symbol.

The result of the puts command is a special value called nil. The nil value represents no value, which is different from a zero value (zero can be a numerical value). You'll see how to use the nil value later on when determining if a statement succeeds or not.

Notice that the test3 value retained the result from the mathematical operation (the 48.36 value). You can now use the test3 variable anywhere else in Ruby to represent that value. To see this in action, enter this code at your IRB prompt:

irb(main):005:0> result = test3 / 2
=> 24.18
irb(main):006:0> puts result
24.18
=> nil
irb(main):007:0>

Another important feature that you can see from this example is that variables retain the data type assigned to them. Since the value in the test3 variable is a floating-point value, the output from the result variable is also a floating-point value.

That's enough on the basics of data types and variables. Follow me to Chapter 4, and let's start writing some Ruby code!



Chapter 4

Creating a Ruby Program

Using IRB to assign values to variables and see the results is fine, but it's not all that practical. The last thing you'll want to do is enter a 1,000-line program by hand every time you want to run it.

Instead, Ruby provides a method for us to store our programs in files and then run them through the Ruby interpreter. The Ruby interpreter program is simply called ruby. It processes Ruby code files and displays any output on the command prompt.

Let's create a simple Ruby program in a file and run it through the Ruby interpreter. Just follow these steps:

  1. Open the InstantRails program as described earlier in Chapter 3, and then open the Ruby Console window.

  2. Create a working folder in the rails_apps folder by typing the command:

    3. mkdir lesson2

  3. Change to the new working folder by typing the command:

    4. cd lesson2

  4. Open Notepad with a new file called first.rb using the command:

    5. notepad first.rb

  5. Enter the following code into the empty Notepad session:


    6. test1 = 10.25
test2 = 5.2
result1 = test1 * test2
puts result1

  6. Save the file, and then exit Notepad.

Warning: If you open Notepad to write your code without specifying a filename and then use the Save As feature to save it, be careful. By default, Notepad places a .txt extension on the end of the file when you save it. To prevent that, you can enclose the filename in double quotes in the Save As text box, like this: "first.rb". This will ensure that Notepad doesn't apply the .txt file extension.

You can now run your Ruby program at the Ruby Console window by using the command:

ruby first.rb

Here's what this should look like:

c:\rails\rails_apps\lesson2> ruby first.rb
53.3
c:\rails\rails_apps\lesson2>

Congratulations, your first Ruby program! Not too exciting, but it did produce our output (and it was even the correct result)!

Debugging Code

Usually programs aren't as simple as the one we just played with. While I'm sure you're an impeccable typist and an even better programmer, at some point you'll probably experience an error in your Ruby code. Fortunately, the Ruby interpreter can help us out.

If there's an error in your code, the Ruby interpreter will point it out and attempt to tell you just what the error is. Let's give this a try:

  1. Open your first.rb code program in Notepad

  2. Change the last line to read:


    3. puts result

  3. Save the file, and exit Notepad.

Now run the new file from the command prompt. Your output should look like this:

C:\rails\rails_apps\lesson2>ruby first.rb
first.rb:4: undefined local variable or method `result' for main:Object (NameErr
or)
C:\rails\rails_apps\lesson2>

The Ruby interpreter produced an error message telling you exactly what file and line the error was found in and what the error is. This can be invaluable in a 1,000-line application.

Using the SciTE Text Editor

Notepad isn't exactly the most advanced editor to use for writing Ruby code. For one thing, it doesn't provide line numbers, so you have no way of knowing what line of code a particular statement is on when you're debugging. It also lacks some of the basic features provided by fancy code editors, such as recognizing Ruby code syntax. So it can't warn you ahead of time if there's an issue with your code.

There are lots of Ruby editors available; some are free, and some are not-so-free. The InstantRails project includes a free code editor that may save you some work (and money).

The Scintilla project is an Open Source project that provides a fancy code-editing component. Scintilla isn't an editor by itself, but it provides the underlying logic for a code editor, giving us SciTE. While the SciTE text editor can't run application code, it does support editing many different programming languages, including Ruby.



The SciTE text editor

The SciTE text editor uses syntax highlighting. This feature uses different colors to highlight keywords, such as Ruby statements and variable names. If you open a code block area, such as a quote delimiter, it'll highlight the item in red so you know instantly if you're missing a quote, instead of having to wait for the error message to appear when you try to run your code. This can be a great time-saver in development work.

The InstantRails application includes SciTE, but it's not obvious how to access it. By default, the editor is located in the ruby/scite folder in the InstantRails installation layout. The name of the executable file is scite.exe.

To use SciTE to edit your Ruby code files automatically, you can associate it with the .rb file extension in Windows Explorer. To do that, just follow these steps:

  1. Start Windows Explorer, and find your first.rb file.

  2. Right-click the first.rb file, and select Open from the menu.

  3. Select the radio button for the Select the program from as list option, and click OK.

  4. Click the Browse button, and then find the sciTE.exe program in your InstantRails installation folder (under the ruby/scite folder).

  5. Check the box to always use this program to open this kind of file, and click the OK button.

Now when you look at your Ruby programs in Windows Explorer, all you need to do is double-click them to start the SciTE editor with the program inside!

Chapter 5

Summary

Today we looked at how Ruby deals with the basic types of data found in our applications. You saw how Ruby uses two different methods for handling numbers. Integer values contain positive and negative whole numbers. These data values are useful for keeping counters and for identifying objects with a unique ID, such as employee IDs.

The floating-point data type incorporates a wide range of numbers by including fractional values. The decimal notation system allows you to specify a fraction to any required decimal place. Ruby also allows you to use scientific notation to represent extremely large or extremely small values.

The string data type provides a method for storing and processing text. You can define string literals using delimiters. A delimiter can be either a single or double quote. You can use the delimiter character inside the string, but you must precede it with a backslash character to identify it as data instead of the delimiter.

Ruby uses variables to store data values in memory while processing a program. Ruby variables can be any length word starting with a lowercase letter and containing both letters and numbers. Once you store a value in a variable, you can retrieve it at any time later in your program, and it'll have the same value (unless, of course, you change the value yourself).

You can use the Interactive Ruby program (IRB) to quickly process Ruby statements in real-time. This allows you to quickly test short code segments without having to write a program.

However, any Ruby statements you enter in the IRB program disappear when you exit. To write a program of any significant length, you'll want to use the Ruby interpreter (ruby). The ruby program allows you to store your code in a file and then process it as many times as you want (or need).

To create your Ruby code file, you'll want to use a text editor. The Notepad editor is adequate to work with, but it doesn't provide many advanced features. The InstantRails package includes a free Ruby text editor called SciTE. The SciTE editor provides some features, such as syntax highlighting and line numbering, which make your life as a Ruby programmer much easier.

In the next lesson, we'll continue our tour of the Ruby programming language. We'll learn how to use Ruby code to manipulate our data and to perform logic to alter the flow of our programs. Until then, don't forget this lesson's quiz and assignment.
